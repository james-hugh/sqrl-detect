## 2026-04-27 - Optimizing C Terminal Output Performance
**Learning:** Significant performance gains (approx. 32% in benchmarks) can be achieved in C CLI applications by reducing the number of `printf` calls. Using static character buffers and the `%.*s` precision specifier allows for printing variable-length repeated characters or substrings in a single call, which is much more efficient than character-by-character loops. Additionally, declaring local string pointer arrays as `static const char* const` prevents redundant stack allocations and initializations on every function call.
**Action:** When rendering CLI components like graphs, meters, or repeating patterns, use static buffers with `%.*s` instead of loops with individual `printf` calls. Always use `static const` for fixed lookup tables or string arrays inside functions.
